# Next Chat Prompt

Use this file when the user says exactly or approximately: "прочитай next-chat".

Meaning of that request: prepare a ready copy-paste onboarding prompt for the next chat of your same role, so the user can archive this chat and continue in the new one.

```text
Ты прочитал `project-memory/prompts/next-chat.md`. Теперь твоя задача - подготовить готовый copy-paste промт для следующего чата той же роли.

Сделай готовый copy-paste промт для следующего чата, который заменит тебя и сможет сразу продолжить работу в той же роли.

Работай от своей текущей роли:

- если ты `GD`, пиши промт для следующего GD;
- если ты `Important`, пиши промт для следующего Important;
- если ты `UI`, пиши промт для следующего UI, без номера;
- если роль другая, сохрани эту же роль и явно назови ее в промте.
- Не добавляй номера к роли: новый чат называется так же, как старый (`Dev` заменяет `Dev`, `UI` заменяет `UI`, `GD` заменяет `GD`).

- Текущий чат будет архивирован или перестанет быть основным.
- Пользователь может отдельно прислать следующему чату последнюю переписку текущего чата, особенно все, что было после последнего push или последнего обновления `project-memory`.

Перед тем как писать промт для следующего чата:
1. Прочитай:
   - `project-memory/README.md`
   - `project-memory/handoff.md`
   - `project-memory/updates.md`
   - `project-memory/prompts/new-chat.md`
2. Прочитай inbox своей роли, если он есть:
   - `Important`: `project-memory/inbox/for-important.md`

   - `GD`: `project-memory/inbox/for-gd.md`

   - состояние относительно `origin/main`.

5. Если какой-то факт неизвестен, явно напиши `неизвестно`, не придумывай.

В промте для следующего чата обязательно укажи:
- Его роль: следующий чат называется так же, как текущая роль, и становится ее активным продолжением.
- Почему создается новый чат: текущий чат уходит в архив/заменяется, а свежая переписка может быть передана пользователем отдельно.

  - `Dev`: разработка и реализация.
  - `GD`: геймдизайн, правила, баланс, core loop, постановки для Dev.

- Краткую текущую форму игры и ключевые файлы, релевантные роли.
- Свежий статус из `project-memory/updates.md`.
- Если релевантно роли, текущий git-статус:
  - ветка;
  - последний локальный коммит;
  - есть ли расхождение с `origin/main`;
  - какие файлы незакоммичены;
  - что уже запушено, а что только локально.
- Что текущий чат делал после последнего push или последнего обновления project-memory, если это известно.
- Что следующему чату нужно сделать первым:
  1. прочитать project-memory;
  2. прочитать переданную пользователем переписку текущего чата;
  3. проверить свой inbox;
  4. познакомиться с Git/GitHub-состоянием проекта;
  5. познакомиться с самой игрой и ее текущими механиками;
  6. если роль работает с файлами, проверить `git status`;
  7. продолжить работу с текущего состояния, не теряя локальные изменения.
- Правила безопасности:

В конце промта для следующего чата добавь блок "Передача следующему чату":
- Если этот чат позже будут заменять на следующий чат той же роли, он должен прочитать `project-memory/prompts/next-chat.md`, собрать свежий project-memory/git/inbox статус и выдать пользователю новый copy-paste промт для продолжателя этой же роли.

После промта для следующего чата отдельно добавь строку для переименования архивируемого чата:
- Формат: `Archive name: <роль/имя чата> - <текущая дата YYYY-MM-DD> - <N дней с создания>`
- Используй свою роль/имя чата без номера, если номер не является официальной ролью.
- Посчитай, сколько календарных дней прошло от создания текущего чата до даты подготовки промта. Если дата создания неизвестна, напиши `N дней: неизвестно`.
- Пользователь переименует текущий чат перед архивацией в этот текст.

Выведи только готовый copy-paste промт для следующего чата в одном блоке `text`.
```
